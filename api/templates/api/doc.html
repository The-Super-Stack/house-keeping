Common

1. "/login" for login endpoint and get the jwt token.

2. "/user" to get the authenticated user.

3."/logout" to kill the token session


Employee Endpoint

1. "/all-employee" its return all registered employees
available query : nik=<nik-like-queries> and ph=<phone_number>

2. "/assignment-control" its return all the employee assignment. use query to get the assignment
available query : wr=<int:user_id>

3. "/qr-validating/<uid>" will checked the assignment is not none. in this endpoint you will send a json to checked it is valid or not
json you must send {"code": "<work_place_qr_decrypted>"}

4. "/assignment-control/list/<uid>" its return the detail of the assignment there is no post query. in this endpoint the only available is POST
so you must send the json like
{
  "id": [**<id>],
  "is_done": [**true||false]
}

5. "/assignment-control/img/<uid>" this endpoint used for upload the img_before and img_after. use PUT method to send the image


Supervisor Endpoint

1. "/supervisor" it shows the list of assignment for the downline(employee/ob)
to display the list, use the available Query
available query : ?me=<int:id>
in this endpoint, you can send post data to the server. please send the json like this :
{
"assignment": assignment-id
"date":" year-month-date",
"time":time-in-minutes-dont-use-single-quote-or-double-quote,
"employee":[**employee-id in list]
}

2. "/supervisor/work-place" only get the data. no post. admin has created. take the id.

3. "/supervisor/all-employee" it returns all the employee not the supervisor. take the employee id. no post.

4. "/supervisor/links" it will show all the owned link by each spv's.
for displaying the data use query parameter. "?spv=<id>" -- like 1 2 or 3
